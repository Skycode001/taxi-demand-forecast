## 📋 Описание проекта

Компания «Чётенькое такси» собирает исторические данные о заказах такси в аэропортах. Цель проекта — построить модель для **прогнозирования количества заказов на следующий час**, чтобы эффективно привлекать водителей в периоды пиковой нагрузки.

### Исходные данные

| Параметр | Значение |
|----------|----------|
| Период | Март – Август 2018 (6 месяцев) |
| Интервал | 10 минут |
| Объём | 26 496 наблюдений |
| Целевая переменная | `num_orders` (количество заказов в час) |

### Постановка задачи

- ✅ Построить модель прогнозирования временного ряда
- ✅ Достичь метрики **RMSE ≤ 48** на тестовой выборке
- ✅ Выявить ключевые факторы, влияющие на спрос

## 🔍 Ключевые выводы

### 📈 Тренды и сезонность

| Компонента | Вывод |
|------------|-------|
| **Тренд** | Рост с 58 → 139 заказов (+81 за полгода) |
| **Суточная сезонность** | Пик в 0:00 (144 заказа), спад в 6:00 (25 заказов) |
| **Недельная сезонность** | Пятница (2187) > Понедельник (2166) > Вторник (1848) |
| **Шум** | Std остатков вырос с 22 → 34 (+55% к августу) |

### 🔑 Важность признаков

| Признак | Важность | Интерпретация |
|---------|----------|---------------|
| `lag_168` | 44.35% | Значение неделю назад |
| `lag_48` | 12.58% | Значение двое суток назад |
| `lag_24` | 11.91% | Значение сутки назад |
| `hour_sin` | 9.31% | Циклический час суток |
| `ewma_6` | 4.01% | Экспоненциальное среднее |

> 💡 **Недельная и суточная память — ключевые факторы (суммарно ~69%)**

### ⚠️ Ошибки модели

| Показатель | Значение |
|------------|----------|
| Средняя ошибка (Mean Error) | 12.41 (модель занижает прогноз) |
| Медианная ошибка | 7.79 |
| 95% интервал ошибок | [-46.36, 90.14] |
| Макс. занижение / завышение | -92 / +233 заказа |

**Причина систематического занижения:** резкий рост спроса в августе (тренд +81 заказ), который модель не успела «выучить» на обучающих данных (март–июль).

## 🚀 Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/your-username/taxi-demand-forecast.git
cd taxi-demand-forecast
```

### 2. Создание виртуального окружения

- macOS / Linux
```bash
python -m venv venv
source venv/bin/activate
```

- Windows
```bash
python -m venv venv
venv\Scripts\activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```
